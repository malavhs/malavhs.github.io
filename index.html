Scramblers -
The Password Folks


Learning Objectives

Students are exposed to some password patterns and receive motivation on finding an effective strategy to come up with passwords for common websites. The students learn what makes a password problematic, and also what makes a password harder to guess. With the help of some worksheets, students will learn how to use an effective password strategy to create strong passwords. By the end of the workshop, students will be able to list the reasons why it is important to maintain secure passwords and also be able to generate secure passwords quickly using the password creation strategy introduced.


Background and Motivation

Cybersecurity is best introduced as a concept when we are younger so that it becomes easier to practice as we grow older. While using simpler passwords or the same password for multiple accounts may be convenient, it is not considered a very safe practice because the creator of the password can compromise multiple accounts’ security simultaneously. Though passwords to store our data should be private, methods to come up with passwords need not be. Instead, they need to be humanly usable and should enable the user to generate all possible passwords. In order to do this, humans can use mindhashes, which require memorizing a certain key or map, to avoid using external sources to generate passwords. Two password strategies that we recommend, additionally leverage a short string of special characters to create unique passwords for different accounts. The first method is the 3-word hash, which requires the creator to memorize three words and a wildcard character, while the second method is the random-letter hash, which requires the creator to memorize a mapping of the alphabet to a random assortment of letters. Based on the previous user studies, the random letter hash requires more time to learn, but participants are able to generate passwords in less amount of time, compared to the 3-word hash.

While the studies mentioned above were done for adults, our group aims to bring awareness of the strategies to kids (middle schoolers) so that they get used to best practices in cybersecurity around the same time that they might start creating their own online accounts. We are exploring libraries as spaces and researching methods to translate information about digital literacy and cybersecurity to children. Our research on existing technologies also helps us design effective pop-up workshops for our target audience.


The Outline

The pop-up workshop will take place in a library and will be around 75 - 90 mins long. The outline or expected flow of the workshop is as follows:

  1. Guess Passwords, a whole class activity (~6 minutes)
  2. Problematic Passwords, a whole class activity (~10 minutes)
  3. Password strategy explanation, a whole class activity (~8-10 minutes)
  4. Teaching How to Remember Code, a whole class activity (~8-10 minutes)
  5. Generate Letter Codes, a small group activity (~12-15 minutes)
  6. Generating password practice, a whole class activity (~5 minutes)
  7. Generate passwords, a small group activity (~10 minutes)
  8. Generate letter maps, an individual take-home activity

F.A.Q

  • Why not just use a password manager?

    Usually password managers require the memorization of one password and also a machine to store the rest of the generated passwords. If the key password is leaked, then all passwords are compromised, so the risk is high. With the ability to generate passwords using a strategy using just our brain, we also don’t need to rely on a single password or a machine to store passwords for all sites.

  • Do you need to change passwords over time?

    This is a good practice, and you could come up with many ways to change the password while still using the same strategy. One way to update the passwords is to use a different word to refer to a website. For example, the word used to generate the password from apple.com could change from apple to orange.

  • How will students remember all of this?

    The memorization strategy we have presented is a result of extensive research that shows that the code generated is effective and hard to forget. Using this password creation strategy also allows the user to practice using the random letter map frequently, and thus improve memory.

  • What age range is this appropriate for?

    This strategy is useful for kids as well as adults. Kids may have an easier time learning and implementing the strategy since they won’t have as many pre-existing passwords to retroactively update, and won’t have to remember a long list of which passwords have or haven’t been updated.

  • What if a student can’t come up with a word for a letter pair?

    Use a word from any language to help you remember the code. Remember that you can have vowels separating the two letters. If this is still hard, just choose a different random letter for your code.

  • Is it okay to use [foreign words, proper nouns, made up words] to remember a code?

    Yes! As long as you personally will be able to remember it, you can use any word you want.

  • How do you handle short website names?

    You might use the website name twice (e.g. appleapple), choose a longer word that makes you think of the website name to generate from (e.g. macintosh for apple.com), add a longer string at the end, or type your entire code words and not just the letters. Whichever strategy you use, just make sure to have a consistent rule for yourself so you can remember!

  • This creates short passwords, which aren’t as secure. Is there a way around that?

    See the above question about short website names! These same strategies can be used to create a longer password for any site.

  • This seems similar to a rotation cypher, which is easy to hack. How is this better?

    In this case, you cannot decode all the letters just by figuring out one or two. It’s still probably better to use a completely unique password for your most important accounts

  • What if the same letter is rolled multiple times?

    That’s fine! It makes it harder to hack the resulting passwords.

  • Why don’t you use vowels as codes?

    It helps with memorization of the code. For example, it might be hard to think of a word to remember the code mapping from b to k without introducing a vowel in between the two. One might choose the word bake to remember this code. If vowels are allowed, you would have to remember whether the code letter was a or k. Only including consonants means that you only have to remember the word itself to know what your code letter is.
